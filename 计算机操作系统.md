- 服务上层应用需求

## OS简史

- 串行调度 
  - 线性完成计算任务

- 机读卡年代
- 批处理 Batch processing
  - 一堆一起处理
  - 中心化节点处理收集到的信息，然后执行
- IO与计算存在矛盾
  - 程序执行的大部分时间在进行读写行为
  - CPU空闲
  - 解决方案
    - 划分流水线
    - 多道编程
    - STI
      - 压缩IO的计算时间相近
      - IO时间与计算时间可以重叠
- 70年代起，CPU原来越快
  - 用户拓宽，平常人也开始用电脑
  - 提升交互性
- 分时系统
  - 时间分片
  - 每个用户、进程分别管理
  - 对多道、批处理，对资源更细粒度的划分和调度
  - “多用户幻觉”：每个用户都感觉自己似乎拥有整台电脑
- 实时系统
  - 在一定时间内，一定有严格数据保证
  - 卫星、精密系统
- OS发展是适应其他条件变化的
  - 新型硬件
  - 新的场景、环境
  - 新的需求

- 现代OS
  - 位置
    - 处于硬件与软件之间
    - 提供双方之间的接口
  - 组成
    - 进程管理
      - 对每个进程进行了线程抽象
      - 地址空间
    - 内存管理
    - 存储管理
